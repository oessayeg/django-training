How Django Displays a Static Page

  1. Client Request: The browser sends an HTTP request (e.g., GET /about/) to the server.
  2. URL Routing: Django checks urls.py to find which view corresponds to the requested path.
  3. View Execution: The matched view function (e.g. about_view) is called. It usually calls render() to combine a template with a context dictionary.
  4. Template Rendering: The template engine loads the HTML file (e.g. about.html), fills it with the given context, and produces a complete HTML page.
  5. Response Creation: Django wraps the rendered HTML in an HttpResponse object.
  6. Middleware Processing: Response middleware can modify or add headers (e.g. security, caching, gzip compression).
  7. Return to Client: The HttpResponse is sent back to the browser, which displays the page.

If the page includes static files (CSS, JS, images), the browser will then send additional requests for those resources, which are served separately.
