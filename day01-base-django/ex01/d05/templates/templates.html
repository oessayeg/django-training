Functioning of Django’s default template engine

Django’s template engine takes an HTML-like template plus a context dictionary and renders a final HTML page. It supports variable substitution, filters, and tags for control flow.

Blocks are placeholders that can be overridden in child templates when using template inheritance, helping to reuse a common layout.

Loops (for ... in) let you iterate over items and also support an {% templatetag openblock %} empty {% templatetag closeblock %} clause when the list is empty.

If control structures let you conditionally display content with {% templatetag openblock %} if {% templatetag closeblock %}, {% templatetag openblock %} elif {% templatetag closeblock %}, and {% templatetag openblock %} else {% templatetag closeblock %}.

Context variables passed from the view are displayed with {% templatetag openvariable %} variable {% templatetag closevariable %}. By default they’re HTML-escaped; you can apply filters like |upper or |default.
